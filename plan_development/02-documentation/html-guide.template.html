This guide documents (), . It is both a system workflow guide and a developer guide. The reader is the development team.

1. Overview — What is

The platform lets a :

The platform has three parts that all share one domain engine:

  • A web app (Next.js) for desktop use.
  • A mobile app (Expo / React Native) for phones.
  • A backend API (.NET) that stores all data and is the system of record.
Key idea: The interactive domain logic runs in the frontend (web and mobile), using a shared TypeScript package @/. The backend stores inputs, results, and scores, and builds the PDF. Domain rules change in the shared package, not in the backend.

6. Backend (Clean Architecture)

The backend follows Clean Architecture with four layers. Dependencies always point inward: outer layers depend on inner ones, never the reverse.

graph LR
  WEB["Web
(Minimal API endpoints)"] --> APP
  INFRA["Infrastructure
(EF Core, MinIO, SMS, Identity)"] --> APP
  APP["Application
(CQRS use cases, validators, mapping)"] --> DOM
  DOM["Domain
(entities, enums, calculators)"]
  WEB --> INFRA
  • Domain — entities, enums, constants, and the domain calculators. No dependencies on other layers.
  • Application — CQRS use cases via MediatR, FluentValidation validators, AutoMapper profiles, and service interfaces. Depends on Domain only.
  • Infrastructure — concrete service implementations, EF Core/SQL Server, ASP.NET Identity. Depends on Application + Domain.
  • Web — Minimal API endpoints. Each IEndpointGroup class is auto-mapped at /api/{ClassName}.

MediatR pipeline behaviors

Every request passes through these cross-cutting behaviors, in order: LoggingBehaviour, UnhandledExceptionBehaviour, AuthorizationBehaviour, ValidationBehaviour, PerformanceBehaviour.

9. Authentication & Roles

There are three sign-in methods. All of them end with the API issuing an Identity bearer token (an access token plus a refresh token). The OTP and Google flows set signInManager.AuthenticationScheme = IdentityConstants.BearerScheme before calling SignInAsync.

Roles: Administrator (manage users + subscriptions, access /api/Admin/*) and User (default). GET /api/Users/me returns { roles, isAdmin }.

Password login

Mobile OTP (one-time password)

  participant S as SMS relay
  C->>A: POST /api/Auth/otp/request (phoneNumber)

14. PDF Reports & File Storage

PDF generation (QuestPDF)

  • QuestPdfReportGenerator.GenerateReport(model) returns a byte[].
  • QuestPDF uses its free community license.

File storage (MinIO)

  • MinioFileStorage implements IFileStorage: PutAsync, GetAsync, DeleteAsync, GetPresignedUrlAsync(key, expiry).
  • The presigned URL is time-limited (1 hour for reports). In production the host is rewritten to the public endpoint so browsers can open it.

16. Local Development Setup

Backend (.NET 10)

# Start SQL Server (sa) + MinIO (minioadmin)
docker compose -f deploy/docker-compose.dev.yml up -d

dotnet build .slnx     # build all
dotnet run --project src/Web         # API on http://localhost:5000 (docs at /scalar)
dotnet test                          # all tests

The dotnet-ef global tool must match EF Core 10: dotnet tool update -g dotnet-ef --version "10.0.*".

Web (Next.js 16)

cd web && npm install
npm run dev      # http://localhost:3000
npm run build    # production build (must pass before deploy)

web/.env.local sets NEXT_PUBLIC_API_BASE (dev http://localhost:5000), baked at build time. Keep output: "standalone" in next.config.ts.

Mobile (Expo)

npm run start      # expo start (Metro dev server + QR)
npm run android    # expo run:android (needs Android SDK)
npm run typecheck
Build is strict: TreatWarningsAsErrors=true. .NET 10 turns deprecations into errors (e.g. use KnownIPNetworks, not KnownNetworks). ValidationException is ambiguous — alias the app's Application.Common.Exceptions.ValidationException. Use Guard.Against.NotFound(...) for 404s.

17. Glossary

TermMeaning
OTPOne-Time Password. A short code sent by SMS to log in.
nvarchar(max)SQL Server's large variable-length text column type. Stores the domain JSON (InputJson/ResultJson).
CQRSCommand Query Responsibility Segregation. Splits writes (commands) from reads (queries). Implemented with MediatR.
Presigned URLA temporary, signed link to a file in object storage (MinIO) that works without login for a limited time.
Bearer tokenAn access token sent in the Authorization: Bearer ... header to prove who the user is.
MinIOAn S3-compatible object storage server. Stores the generated PDF reports.
QuestPDFA .NET library used to build the PDF reports.
TraefikA reverse proxy that routes traffic to the web, API, and S3 domains and handles TLS.
DenormalizeStoring a value (like TotalScore) in its own column even though it could be computed, so queries are faster.

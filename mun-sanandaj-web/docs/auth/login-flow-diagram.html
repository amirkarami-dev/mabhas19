🔐 Login Flow — mun-sanandaj-web

How a visitor goes from a locked page to the Dashboard, using OIDC. Hover a step to highlight it.

Our app (React) Central login server (IdP) Guard / redirect
Step 1 · Browser
Visitor opens /
Home is a protected page — it is wrapped in the <RequireAuth> guard.
Step 2 · Guard
RequireAuth checks for a user
No logged-in user found → redirect to /login.
(While still checking, it shows a spinner because ready is false.)
no user
Step 3 · Screen
LoginScreen — click “ورود”
One button. It calls login(), which runs signinRedirect().
signinRedirect()
↓ browser leaves our app ↓
Steps 4–5 · IdP
User signs in on the login server
Username / password / OTP — handled entirely by the IdP. Our app never sees the password. On success the IdP sends the browser back to redirect_uri.
redirect to /auth/callback
Step 6 · Screen
OidcCallback saves the token
signinRedirectCallback() reads the code from the URL, swaps it for a token, and stores it in localStorage. Then navigates to /.
userLoaded event
Step 7 · State
AuthProvider sets the user
It hears the userLoaded event and updates its state. Now useAuth() returns a real user.
has user ✓
Step 8 · Guard
RequireAdmin checks the role
Is isAdmin true? The answer splits the flow in two:
Admin ✓
Dashboard
Login complete. User sees the app.
Not admin ✗
/403 — Forbidden
ForbiddenScreen is shown.
